### 1. Problem Statement
OpenClaw-RL addresses the lack of agentic RL systems that recover **next-state signals**—the user reply, tool output, terminal result, or GUI state transition that follows action \(a_t\)—as a **live, online learning source**. Existing systems treat next-state information as passive context rather than actionable RL supervision, even though it implicitly contains (i) how well \(a_t\) performed and (ii) how it should have changed.

- **Waste 1: Evaluative signals**  
  Next-state signals often implicitly score the preceding action (e.g., user re-query = dissatisfaction; test pass/fail = success/failure), forming dense **process reward** candidates for RL credit assignment. Existing PRM usage in RLVR is largely confined to settings with verifiable offline ground truth or outcome-only rewards, discarding turn-level evaluative density.

- **Waste 2: Directive signals**  
  Next-state signals frequently contain explicit correction direction (e.g., “you should have checked the file first”; detailed SWE error traces that imply what to change). Existing RLVR uses scalar rewards and cannot convert this directional content into **token-level directional policy updates**, while offline distillation/hindsight methods rely on pre-collected feedback-response pairs or implicit hints that are not recovered explicitly from live next-state.

**Core Challenge:** Recover both evaluative and directive information from heterogeneous next-state signals across interaction types, and use it for **continuous online policy improvement** with minimal serving/training coordination.

### 2. Methodology
**OpenClaw-RL** is a unified framework that performs continuous RL training for personal and general agents by extracting learning signals from **next-state signals** in an asynchronous, fully decoupled server architecture.

#### A. System Architecture: Asynchronous Four-Loop Decoupling
**Mechanism:** OpenClaw-RL uses a full decoupling design where policy serving, environment hosting/rollout collection, PRM judging, and policy training run as **four independent asynchronous loops** with no blocking dependencies.

**Process:**
1. **Policy Serving loop (SGLang):** serves the next user request using current policy.
2. **Environment loop (HTTP / API):** returns next-state signal \(s_{t+1}\) (user reply, tool output, terminal/GUI transition).
3. **PRM/Judge loop (SGLang / API):** judges the previously served action \(a_t\) using \(s_{t+1}\), producing reward components.
4. **Training loop (Megatron):** applies gradient updates concurrently while serving continues.

**Scope/Role:** Enables **online training from live deployment** and supports heterogeneous streams (personal and general) without pausing inference.

#### B. Unified Problem Setting as MDPs with Next-State Reward Extraction
**Mechanism:** Each interaction stream is formalized as an MDP \((S, A, T, r)\), where next-state is deterministically produced by the environment given the action.

**Process:**
1. **State:** \(s_t \in S\) = full conversational/environment context up to turn \(t\).
2. **Action:** \(a_t \in A\) = agent response tokens generated by \(\pi_\theta\).
3. **Transition:** \(T(s_{t+1}\mid s_t, a_t)\) deterministic; \(s_{t+1}\) is user reply / execution output / next GUI state / verdict.
4. **Reward:** \(r(a_t, s_{t+1})\) inferred from the next state via a **PRM judge** (binary score and/or derived hints).

**Scope/Role:** Makes reward computation **process- and turn-dependent**, enabling dense credit assignment instead of trajectory-only outcome rewards.

#### C. Personal-Agent Optimization: Two Complementary Methods + Shared PPO Loss
OpenClaw-RL supports personal agents with (i) **Binary RL** from evaluative signals and (ii) **Hindsight-Guided OPD** from directive signals, combined via a weighted advantage.

---

##### C1. Binary RL (Evaluative Signals → Scalar Process Reward)
**Mechanism:** A PRM judge maps \((a_t, s_{t+1})\) to a discrete reward \(r\in\{+1,-1,0\}\), used as an advantage in PPO-style training.

**Process:**
1. **PRM evaluation:**  
   \[
   \text{PRM}(a_t, s_{t+1}) \rightarrow r \in \{+1,-1,0\}.
   \]
2. **Majority vote aggregation:** run \(m\) independent PRM queries and compute  
   \[
   r_{\text{final}} = \text{MajorityVote}(r_1,\dots,r_m).
   \]
3. **Advantage assignment:** set \(A_t = r_{\text{final}}\).
4. **PPO clipped surrogate with asymmetric bounds + KL penalty:**
   \[
   \rho_t=\frac{\pi_\theta(a_t\mid s_t)}{\pi_{\text{old}}(a_t\mid s_t)},
   \]
   \[
   L_{\text{pg}}=-\mathbb{E}_t\left[\min\left(\rho_t A_t,\ \text{clip}(\rho_t,1-\varepsilon,1+\varepsilon_{\text{high}})\cdot A_t\right)\right],
   \]
   \[
   L = L_{\text{pg}} + \beta_{\text{KL}} L_{\text{KL}},
   \]
   with \(\varepsilon=0.2\), \(\varepsilon_{\text{high}}=0.28\), \(\beta_{\text{KL}}=0.02\).

**Scope/Role:** Learns from any next-state signals (re-queries, tool results, terse reactions) when directive extraction is unreliable or unavailable.

---

##### C2. Hindsight-Guided On-Policy Distillation (OPD) (Directive Signals → Token-Level Direction)
**Mechanism:** Convert directive content in next-state \(s_{t+1}\) into **token-level advantage supervision** by extracting actionable textual hints and using them to build an enhanced teacher context.

**Process (token-level OPD):**
1. **Judge for hint extraction:**  
   \[
   \text{Judge}(a_t, s_{t+1}) \rightarrow \{(\text{score}_i,\ \text{hint}_i)\}_{i=1}^m,
   \]
   where \(\text{score}\in\{+1,-1\}\).
2. **Quality filtering:** keep only hints satisfying \(\text{score}=+1\) and \(|\text{hint}|>10\) characters; if none exist, **drop the sample**.
3. **Hint selection:** choose the longest valid hint \( \text{hint}=\arg\max |\text{hint}|\).
4. **Enhanced teacher context:** append hint to last user message:  
   \[
   s_{\text{enhanced}} = s_t \oplus \text{"[user’s hint]\textbackslash n\{hint\}"}.
   \]
5. **On-policy distillation via per-token log-prob gap:** query the policy model under \(s_{\text{enhanced}}\) with the original response \(a_t\) as forced input to compute token log-probabilities, and define token-level advantage:
   \[
   A_t = \log \pi_{\text{teacher}}(a_t\mid s_{\text{enhanced}}) - \log \pi_\theta(a_t\mid s_t).
   \]
   Here \(A_t>0\) indicates the student should upweight tokens, and \(A_t<0\) indicates downweighting.

**Scope/Role:** Uses directive next-state information that scalar rewards cannot express, producing richer token-level directional gradients.

---

##### C3. Combine Binary RL + OPD via Weighted Advantage
**Mechanism:** Both methods use the same PPO objective; only advantage computation differs. They are combined into a single advantage:
\[
A_t = w_{\text{binary}}\, r_{\text{final}} + w_{\text{opd}}\left(\log \pi_{\text{teacher}}(a_t\mid s_{\text{enhanced}})-\log \pi_\theta(a_t\mid s_t)\right),
\]
with \(w_{\text{binary}}=w_{\text{opd}}=1\) by default.

**Scope/Role:** Maintains coverage (Binary RL accepts all scored turns) while injecting high-resolution correction direction where hints are extractable (OPD).

---

#### D. General-Agent Optimization: Step-wise Process Rewards + Outcome Rewards Integration
**Mechanism:** For long-horizon agentic tasks (terminal, GUI, SWE, tool-call), OpenClaw-RL uses PRM to produce **dense step-wise process reward** \(r_i \sim \text{PRM}(a_t,s_{t+1})\) for each step and integrates with verifiable outcomes.

**Process/Reward Integration:**
- Outcome rewards are added to step-wise process rewards; step reward is defined as “outcome + mean process rewards,” stated as:
  \[
  r_t = o + \frac{1}{m}\sum_{i=1}^{m}\frac{r_i}{m}
  \]
  (paper describes using \(o +\) averaged PRM outputs per step; implementation follows the “outcome + mean(prm)” step-wise scheme).
- Advantages are standardized by grouping actions with the **same step index** (instead of state clustering, since real-world states are hard to cluster).

**Scope/Role:** Provides dense credit assignment beyond terminal-only outcome rewards for general agentic RL.

### 3. Key Contributions
- **Live recovery of next-state signals as RL supervision:** Identifies next-state signals as containing both **evaluative** and **directive** information and recovers them online across personal and general interaction types.
- **Unified asynchronous architecture for continuous online RL:** Implements a fully decoupled four-loop design (serving, environment, PRM judging, training) enabling zero-serving interruption with graceful weight updates.
- **Binary RL from evaluative next-state signals:** Uses PRM majority-voted discrete judgments \(r\in\{+1,-1,0\}\) as scalar process rewards in a PPO-style clipped surrogate.
- **Hindsight-Guided OPD for token-level directional learning:** Extracts textual hints from next-state, builds an enhanced teacher context, and computes token-level advantages via per-token log-prob gaps.
- **Integrated process + outcome rewards for general agents:** Demonstrates the importance of dense step-wise PRM rewards in long-horizon terminal/GUI/SWE/tool-call settings.

### 4. Experimental Results
#### Track 1: Personal Agents (Conversational Signals)
**Setup:** Personal-agent simulation with **student** and **teacher** both implemented as LLMs operating on personal computer contexts. Homework tasks from **GSM8K**; the OpenClaw policy model is **Qwen3-4B**. Training uses immediate updates after every 16 samples for the simulated runs (learning rate \(1\times 10^{-5}\), KL coefficient \(0\), as specified).

**Findings:**
- **Binary RL vs OPD vs Combined (GSM8K personalization score):** base score **0.17**.  
  - Updated **8 steps**: Binary RL **0.25**, OPD **0.25**, Combined **0.76**  
  - Updated **16 steps**: Binary RL **0.23**, OPD **0.72**, Combined **0.81**
- **Learning speed (interaction counts):** under **combined** optimization, OpenClaw needs **36 problem-solving interactions** (student setting) and **24 grading interactions** (teacher setting) to show significant and clearly visible improvement (Figure 2).
- **Observed behavior changes:** after 36 student interactions, the model reduces obviously AI-like phrasing (e.g., “bold” and overly structured step-by-step style); after 24 teacher interactions, it writes feedback that becomes **friendlier and more detailed**.

#### Track 2: General Agents (Terminal, GUI, SWE, Tool-Call)
**Setup:** Models: **Qwen3-8B**, **Qwen3VL-8B-Thinking**, **Qwen3-32B**, and **Qwen3-4B-SFT** (fine-tuned per Zhu et al. 2025 on Feng et al. 2025a). Datasets: **SETA RL data**, **OSWorld-Verified**, **SWE-Bench-Verified**, **DAPO RL data**. Metrics: average rollout-task accuracy over windows of RL steps for terminal and SWE; GUI uses training-set evaluation (excluding chrome and multi-apps tasks); tool-call uses evaluation on **AIME 2024**.

**Findings:**
- **Scalability via environment parallelization:** uses **128** parallel environments for terminal, **64** for GUI and SWE, and **32** for tool-call during RL training (Figure 4).
- **Outcome + process reward integration improves performance** (Table 4):
  - **Tool-call:** integrated **0.30** vs outcome-only **0.17**
  - **GUI:** integrated **0.33** vs outcome-only **0.31**
- **Process rewards trade-off:** integrating process rewards improves performance but increases resource needs because PRMs must be hosted.

### 5. Significance and Impact
- **Paradigm shift in agent RL data sources:** Next-state signals become an explicit **online learning stream** rather than discarded context, enabling continual improvement from deployment interactions.
- **Architectural novelty for continuous training:** The four-loop **asynchronous, fully decoupled** slime-based design supports live serving during training with graceful weight updates, avoiding coordination bottlenecks.
- **Directional supervision beyond scalar rewards:** **Hindsight-Guided OPD** turns directive next-state information into **token-level advantage**, addressing the limitation of scalar RL rewards in corrective feedback scenarios.
- **Unified RL across heterogeneous environments:** The same pipeline supports personal conversations and general agentic settings (terminal, GUI, SWE, tool-call) using stream-agnostic next-state recovery.
- **Dense credit assignment for long-horizon tasks:** Demonstrates that step-wise **process rewards** via PRM are critical when outcomes occur only at the terminal step.

**In summary:** OpenClaw-RL treats next-state signals as universal evaluative+directive supervision, recovers them with PRM judging and hindsight-guided OPD, and trains continuously in a fully decoupled asynchronous architecture across both personal and general agentic RL settings.
